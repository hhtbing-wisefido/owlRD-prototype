# 核心工作原则

## 🔴 最高优先级规则

### 1. 禁止创建临时文档
- **严禁**在 `.windsurf/rules/` 创建任何临时/分析文档
- 临时文档放 `项目记录/7-过程记录/`
- 分析报告放 `项目记录/6-开发规范/规则系统分析/`

### 2. 严格目录控制
- **只能**创建架构模板定义的目录
- **禁止**随意新增目录或改变目录结构
- 参考 `06-directory-architecture-template.md`

### 3. 规则自检机制
- 每次操作后**必须**自检是否符合规则
- 发现违规立即纠正
- 参考 `08-rule-self-enforcement.md`

### 4. 文档同步
- 修改文件后**必须**更新相关索引
- 保持README和实际内容一致
- 参考 `05-change-synchronization.md`

### 5. 知识库只读
- **禁止**修改知识库内容
- 知识库仅供参考
- 新内容放项目记录

## 优先级顺序
1. 🔴 核心原则（本文件）
2. 🔴 严格目录控制 (07)
3. 🔴 规则自检 (08)
4. 🟡 其他强制规则 (01-05)
5. 🟢 参考文档

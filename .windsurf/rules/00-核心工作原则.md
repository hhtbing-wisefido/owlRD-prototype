# 核心工作原则

## 🔴 最高优先级规则

### 1. 禁止创建临时文档
- **严禁**在 `.windsurf/rules/` 创建任何临时/分析文档
- 临时文档放 `项目文档/`
- 分析报告放 `项目文档/`

### 2. 严格目录控制
- **只能**创建架构模板定义的目录
- **禁止**随意新增目录或改变目录结构
- 参考 `09-项目结构与配置.md`

### 3. 规则自检机制 ⭐
每次操作后**必须**立即自检：

**自检流程**：
1. ✅ 检查是否创建了规范外的文件/目录
2. ✅ 检查文件是否放在正确位置
3. ✅ 检查是否违反命名规范
4. ✅ 检查是否需要同步文档
5. ✅ 发现违规立即纠正

**自检清单**：
- [ ] 文件位置正确？
- [ ] 文件命名符合规范？
- [ ] README已更新？
- [ ] 索引已同步？
- [ ] 目录结构符合模板？

### 4. 文档同步
- 修改文件后**必须**更新相关索引
- 保持README和实际内容一致
- 参考 `04-变更同步规则.md`

### 5. README维护
- 每个目录的README必须保持最新
- 文件变动时立即更新README
- 参考 `05-README管理规范.md`

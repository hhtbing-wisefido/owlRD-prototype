# 项目维护清单

**目的**: 保持项目文档规范、代码质量和结构清晰  
**频率**: 每周检查一次，重大更新后立即检查  
**创建时间**: 2025-11-22  

---

## 📋 每日维护任务

### ✅ 1. 更新聊天记录（如有开发活动）
- [ ] 检查是否有新的开发会话
- [ ] 创建或更新当日聊天记录：`8-聊天记录/YYYY-MM-DD_完整对话记录.md`
- [ ] 记录：主要目标、完成内容、技术亮点、Git提交

**触发条件**:
- 完成一个Phase或重要功能
- 解决重要问题
- 会话即将结束

**模板位置**: `8-聊天记录/2025-11-22_完整对话记录.md`（作为参考）

---

### ✅ 2. 检查临时文件
- [ ] 运行自动检查：`scripts\check_standards.bat`
- [ ] 查看是否有新的临时文件或进度报告
- [ ] 及时归档已完成的临时文档

**需要关注的文件名关键词**:
- "临时"、"进度"、"待办"、"TODO"
- "URGENT"、"紧急"
- "测试"、"test"

**处理方式**:
- 进度报告 → 完成后移到 `7-过程记录/`
- 已解决问题 → 移到 `1-归档/`
- 临时测试文件 → 删除或归档

---

## 🔧 维护工具

### 自动化脚本
| 脚本 | 功能 | 运行方式 |
|------|------|----------|
| check_directory_standards.py | 目录规范检查 | `python scripts/check_directory_standards.py` |
| check_standards.bat | 便捷检查脚本 | 双击运行 |
| update_project_status.py | 更新项目状态 | Git hook自动运行 |
| validate_alignment.py | 后端对齐验证 | `python backend/scripts/validate_alignment.py` |
| validate_frontend_types.py | 前端类型验证 | `python backend/scripts/validate_frontend_types.py` |
| sync_checklist.py | 同步检查清单 | `python backend/scripts/sync_checklist.py` |

### 快捷命令
```bash
# 目录规范检查
scripts\check_standards.bat

# 项目状态更新
scripts\update_status.bat

# 完整验证
cd backend
python scripts/sync_checklist.py && python scripts/validate_frontend_types.py
```

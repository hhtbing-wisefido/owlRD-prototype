## 🗂️ 目录结构定义

### 文档目录

```
[文档目录名]/
├── README.md                    # 文档索引
│
├── 1-[分类名]/                  # 第一类文档
│   ├── README.md
│   └── ...
│
├── 2-[分类名]/                  # 第二类文档
│   ├── README.md
│   └── ...
│
├── 3-[分类名]/                  # 第三类文档
│   ├── README.md
│   └── ...
│
└── ...
```

**说明**：
- 根据项目需要定义文档分类
- 建议使用编号前缀保持顺序
- 每个分类目录应有 README.md 说明

### 代码目录

```
[代码目录名]/
├── [后端目录]/                  # 后端代码
│   ├── [主要代码目录]/
│   ├── [配置目录]/
│   └── README.md
│
├── [前端目录]/                  # 前端代码
│   ├── src/
│   ├── public/
│   └── README.md
│
├── tests/                       # 测试文件
│   └── README.md
│
└── README.md
```

**说明**：
- 根据实际项目结构填写
- 保持代码与文档分离
- 每个主要目录应有 README.md

## 📋 文件位置判断流程

### 创建文件时如何判断位置？

```
问题1: 这是什么类型的文件？
├─ 代码文件
│  └─ 放在 [代码目录] 下
│     ├─ [后端] → [后端目录]/
│     ├─ [前端] → [前端目录]/
│     └─ 测试 → tests/
│
├─ 文档文件
│  ├─ 问题1.1: 是什么类型的文档？
│  │  ├─ [分类1] → [文档目录]/1-[分类名]/
│  │  ├─ [分类2] → [文档目录]/2-[分类名]/
│  │  └─ ...
│  │
│  └─ 问题1.2: 是否是顶层README？
│     ├─ 项目总README → 根目录/README.md
│     └─ 子目录README → 对应子目录/README.md
│
├─ 配置文件
│  ├─ [配置类型1] → [位置1]
│  └─ [配置类型2] → [位置2]
│
└─ 脚本文件
   ├─ [脚本类型1] → [位置1]
   └─ [脚本类型2] → [位置2]
```

**说明**：根据项目实际情况调整此流程图

## 🔍 索引更新规则

### 创建文件后必须更新索引

**1. 在文档目录创建文档**：
   - 更新 `[文档目录]/README.md` 的索引
   - 添加到对应分类

**2. 在代码目录创建重要文件**：
   - 更新对应的 README.md
   - 如是新功能，更新项目状态文件（如有）

**3. 创建新的子目录**：
   - 在父目录的 README.md 中添加说明
   - 在新目录中创建 README.md

---

## 🚫 严格禁止的操作

### 绝对不允许

1. ❌ **在项目根目录创建任何文档文件**
   - 所有文档必须放在文档目录下

2. ❌ **在文档根目录随意创建文件**
   - 必须放在分类目录中

3. ❌ **使用临时、模糊的文件名**
   - 必须遵循命名规范

4. ❌ **跳过检查直接创建文件**
   - 必须先搜索是否已存在
   - 必须确认正确位置

5. ❌ **创建文件后不更新索引**
   - 必须更新 README.md

## 🔧 配置步骤

### 如何使用此模板

1. **复制模板**
   ```bash
   cp project-config.example.md project-config.md
   ```

2. **填写项目信息**
   - 替换所有 `[占位符]` 为实际内容
   - 删除不适用的部分
   - 添加项目特定的规则

3. **定义目录结构**
   - 画出项目的实际目录树
   - 说明每个目录的用途
   - 定义文件应该放在哪里

4. **制定项目规则**
   - 根目录清洁规则
   - 文件命名规则
   - 特定的约束和要求

5. **验证配置**
   - 在新对话中测试规则是否生效
   - 让AI创建文件，检查是否遵守规则
